Amazon recently introduced the **Nova** model family to AWS Bedrock. For enterprise developers coming from heavily regulated environments like Canadian Banking, adopting a new foundation model requires strict adherence to security and compliance protocols.

In this guide, I will demonstrate how to securely invoke the **Amazon Nova Lite** model using the `Converse API`, integrating Bedrock Guardrails to prevent data leakage and prompt injection.

---

## 1. The Bedrock Converse API

The `Converse` API is AWS Bedrock's unified interface for interacting with foundation models. It standardizes the request payload across different models (Claude, Nova, Titan, etc.), making it easier to switch models without rewriting your application logic.

### Prerequisites

You need `boto3` installed and configured with an IAM role that has the `bedrock:InvokeModel` permission.

```bash
pip install boto3
```

## 2. Basic Implementation with Python

Here is a clean, production-ready implementation for invoking `amazon.nova-lite-v1:0`.

```python
import boto3
import json

def invoke_nova_lite(prompt: str) -> str:
    # Initialize the Bedrock Runtime client in your preferred region
    client = boto3.client("bedrock-runtime", region_name="us-east-1")
    
    # Define the model ID
    model_id = "amazon.nova-lite-v1:0"
    
    # Structure the message payload
    messages = [{
        "role": "user",
        "content": [{"text": prompt}]
    }]
    
    # Define inference parameters
    inference_config = {
        "temperature": 0.5,
        "maxTokens": 1024,
        "topP": 0.9
    }
    
    try:
        # Call the Converse API
        response = client.converse(
            modelId=model_id,
            messages=messages,
            inferenceConfig=inference_config
        )
        
        # Extract the generated text
        output_message = response['output']['message']['content'][0]['text']
        
        # Extract Token Usage for cost tracking
        token_usage = response['usage']
        print(f"Token Usage: Input {token_usage['inputTokens']}, Output {token_usage['outputTokens']}")
        
        return output_message

    except Exception as e:
        print(f"Error invoking Bedrock: {e}")
        return ""

# Example usage
print(invoke_nova_lite("Explain the concept of Zero Trust Architecture."))
```

## 3. Securing the Invocation with Bedrock Guardrails

In a banking environment, you cannot trust the user's prompt, and you cannot implicitly trust the model's output. We must apply **Amazon Bedrock Guardrails**.

Assume we have already created a Guardrail in the AWS Console (e.g., ID `gr-123456789abc`, Version `1`) that blocks PII (Personally Identifiable Information) and filters out Prompt Injection attempts.

We can apply this guardrail dynamically within the `Converse` API call.

```python
    try:
        # Call the Converse API with Guardrails Enabled
        response = client.converse(
            modelId=model_id,
            messages=messages,
            inferenceConfig=inference_config,
            guardrailConfig={
                "guardrailIdentifier": "gr-123456789abc",
                "guardrailVersion": "1",
                "trace": "enabled"
            }
        )
        
        # Check if the Guardrail intervened
        if response['stopReason'] == "guardrail_intervened":
            print("SECURITY ALERT: Guardrail blocked the request/response.")
            return "This request violates enterprise security policies."
            
        return response['output']['message']['content'][0]['text']

    except Exception as e:
        # Handle exceptions...
```

## 4. System Prompts and Role-Based Access

When building Agentic AI, the system prompt defines the boundary of the AI's persona. The Converse API supports injecting system prompts cleanly.

```python
    system_prompts = [{"text": "You are a strict cybersecurity compliance officer. You must never provide code that bypasses authentication mechanisms."}]
    
    response = client.converse(
        modelId=model_id,
        messages=messages,
        system=system_prompts,
        inferenceConfig=inference_config
    )
```

## Conclusion

The Amazon Nova models provide excellent performance-to-cost ratios on AWS Bedrock. However, taking them to production requires rigorous enforcement of security boundaries. By utilizing the Converse API combined with Bedrock Guardrails and carefully scoped System Prompts, you can safely deploy these models into highly regulated enterprise environments.
